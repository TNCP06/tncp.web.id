Notes:
- `apps/web` is `"type": "module"` (Payload CLI needs ESM).
- No top-level `app/layout.tsx` on purpose: `(frontend)` and `(payload)` are separate root layouts (Payload requirement). `robots.ts` must live at `app/` root (route-group placement did not register); `sitemap.ts` works inside `(frontend)`.
- `importMap.js` is generated by Payload — do not hand-edit.
- Data reads are wrapped in `unstable_cache` with tags (`profile`, `portfolio-entries`, `portfolio-entry:<slug>`); Payload `afterChange`/`afterDelete` hooks bust them. Revalidation is skipped when Payload runs outside a request (seed/CLI).

Update this table whenever the structure changes.
